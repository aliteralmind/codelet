Current release: Version 0.1.4.2

Build logs, permalink

Code:

  • Upgraded XBN-Java to 0.1.5.1
  • Upgraded TemplateFeather to 0.1.1.3

Build:
  • Changed jar file creation to not delete old jars. Instead, they are updated. This prevents Windows from randomly locking these files, and therefore randomly crashing the builds, because they can't be deleted. (They can be manually deleted, but not automatically. See http://stackoverflow.com/questions/77436/ant-is-not-able-to-delete-some-files-on-windows

Version 0.1.3

permalink

All downloads: http://aliteralmind.com/docs/computer/programming/codelet/codelet-0.1.3/

Notes:

  • Changed codelet-*-core.jar to codelet-*.jar
  • Now has a POM for both deploying jars to Maven Central, and so other projects can use Codelet via Maven Central.
  • {@.file.textlet} (see com/github/aliteralmind/codelet/CodeletType.html#FILE_TEXT):
    • ADDED: Ability to prefix path with an environment variable or system property. Example: "$<base_path>/subdir/input.txt". (Idea from Stack Overflow user Unihedron.)
    • ADDED/CHANGED: Relative paths containing single url slashes ('/') as its file separator, are automatically changed to the appropriate separator for the underlying OS. Absolute paths are unaltered.
